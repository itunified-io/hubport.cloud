3. How Auto-Mapped + Manual Roles Combine

Source Storage Removable?
Congregation flag (auto) FLAG_TO_APP_ROLE lookup at login ❌ Only by changing the congregation record
Manual assignment AppRoleMember DB record ✅ Remove via × button on Roles tab

Permission merge: A publisher's effective permissions = union of all assigned roles (auto + manual). If someone has Territory Servant from flag AND Service Overseer Assistant manually, they get permissions from both. No conflicts — permissions are additive.

4. API

Endpoints for manual role assignment
// List roles assigned to a publisher
GET /publishers/:id/roles
Returns: { autoMapped: [...], manual: [...] }
// Assign a role manually
POST /publishers/:id/roles
Body: { roleId: uuid }
Creates AppRoleMember record
// Remove a manual role
DELETE /publishers/:id/roles/:roleId
Only works for manual roles — auto-mapped roles return 409
// List all available roles (for dropdown)
GET /roles
Returns all seed roles with name, description, permissions count

5. Permission Gating

Who can manage roles: Only users with app:publishers.edit permission see the Roles tab and can assign/remove roles. This is typically elders and the service overseer. Regular publishers see their own roles (read-only) in their profile but cannot modify them.
