2. Bug Fix #4 — Wire "New Territory" Creation

Creation flow — what happens when "+ New Territory" is clicked
1 Click "+ New Territory" → modal dialog: enter number + name
2 POST /territories with number + name (no boundaries yet) → territory created
3 Navigate to /territories/:newId → TerritoryDetail opens
4 Auto-enter CreationFlow (draw mode) since no boundary exists
5 User draws polygon (click-to-place or shift+drag lasso)
6 User closes polygon → auto-fix preview → accept → saves with version v1

Key decision: The button on map view creates the territory record first (number + name only), then redirects to the detail page where CreationFlow activates automatically for boundary drawing. This avoids building a full drawing mode into the map view page. The "no boundary" state in TerritoryDetail already shows a placeholder — we replace that with a "Draw Boundary" button or auto-enter draw mode.

3. Auto-Fix Preview Dialog

Preview overlay when auto-fix modifies geometry
🗺️ Map showing both polygons overlaid
Original
After auto-fix
Boundary will be adjusted:
  • Clipped to congregation boundary (removed 340 m² outside)
  • Removed overlap with #302 Iffeldorf (120 m²)

Behavior: This dialog only appears when geometryModified: true. If the polygon has no violations (doesn't exceed boundary, no overlaps), it saves directly with no preview — instant save. The preview shows both polygons overlaid on the map so the user can visually confirm the changes.

Interaction: Click a violation badge → navigates to that territory's detail page where the user can click "Edit" → make adjustments → save triggers auto-fix preview. The GET /territories/violations endpoint is called once on map load. Red badge = exceeds congregation boundary. Amber badge = overlaps another territory.
