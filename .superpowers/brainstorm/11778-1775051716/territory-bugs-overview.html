Territory Polygon — 4 Bugs to Fix

Based on your screenshot and description. Which bugs are highest priority?

1

🔴 Polygon exceeds congregation boundary

Some territory polygons extend beyond the dashed red congregation boundary line. The spec defines auto-clipping via ST_Intersection() on save — but it's not implemented. Current PUT /territories/:id is a plain Prisma update with zero spatial validation.

Spec says

  • Auto-clip to congregation boundary on save
  • Return autoFix.applied: ["congregation_clip"] in response
  • Toast: "Boundary adjusted — clipped to congregation area"

Current state

  • No PostGIS helpers exist
  • No spatial SQL at all in save pipeline
  • Boundaries stored as raw JSONB, never validated
2

🟡 Territory polygons overlap each other

Adjacent territories share area — their polygons overlap instead of sharing clean edges. The spec defines overlap detection (not auto-fix) with warning badges showing overlap area.

Spec says

  • Detect overlaps after save via ST_Intersects()
  • Return autoFix.overlaps: [{territoryId, number, overlapAreaM2}]
  • Yellow warning badge: "Overlaps with T-5 (121 m²)"
  • Do NOT auto-resolve — user adjusts manually

Current state

  • No overlap detection
  • No visual feedback for overlapping areas
  • Snap engine helps prevent overlaps but doesn't enforce
3

🔴 Edit polygon not working

Vertex handles don't appear when clicking a territory on the detail page. The TerritoryEditor, VertexHandle, and MidpointHandle components exist in code but may not be wired up correctly or the edit mode entry point is broken.

What exists

  • TerritoryEditor.tsx with full vertex manipulation
  • VertexHandle + MidpointHandle components
  • SnapEngine with 4-level priority
  • Undo/redo hook

What's likely broken

  • Edit mode activation (button/click handler)
  • Map event wiring for vertex drag
  • Component rendering within detail page
4

🔴 New Territory creation not working

The "+ New Territory" button in map view doesn't produce a working territory. The CreationFlow component exists with click-to-place and freehand lasso modes, but the POST /territories endpoint is a minimal Prisma create — may not be handling boundaries correctly.

What exists

  • CreationFlow.tsx with 2 drawing modes
  • POST /territories endpoint
  • Territory number + name input

What's likely broken

  • API saves territory but boundaries may not persist
  • Drawing mode may not activate on map
  • No spatial validation on create

Implementation Gap Summary

The Drawing UX spec (Spec 1) designed a complete auto-fix pipeline with PostGIS spatial validation, but the current backend is a plain CRUD API. The entire spatial layer needs to be built:

  • postgis-helpers.ts — doesn't exist yet (ST_MakeValid, ST_Intersection, ST_Intersects)
  • Auto-fix pipeline — validate → water clip → congregation clip → overlap detect
  • autoFix response — extended PUT response with fix metadata
  • Frontend feedback — toasts for auto-fixes, warning badges for overlaps
