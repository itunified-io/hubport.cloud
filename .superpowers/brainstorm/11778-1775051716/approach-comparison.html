Implementation Approach

Two approaches for building the spatial validation + polygon fix system. I recommend Approach A.

A

🟢 Recommended: PostGIS Raw SQL Pipeline

Build postgis-helpers.ts with raw SQL using Prisma's $queryRawUnsafe. All spatial operations (validate, clip, overlap detect) run as a single SQL transaction on the server. Preview is a dry-run endpoint that returns the proposed geometry without saving.

Pros

  • PostGIS is already installed (the DB uses postgis/postgis:16-3.4-alpine)
  • ST_MakeValid, ST_Intersection, ST_Difference are battle-tested
  • Single SQL transaction = atomic, no partial state
  • Consistent with the existing Drawing UX spec design
  • Server-authoritative — no client-side geometry manipulation needed
  • Fast — PostGIS handles complex polygon operations in <100ms

Cons

  • Raw SQL outside Prisma's type system (mitigated with typed wrapper functions)
  • Boundaries stored as JSONB, need ST_GeomFromGeoJSON conversion on every query
Architecture:
POST /territories/:id/preview-fix → dry-run auto-clip, return diff
PUT /territories/:id → save with auto-fix pipeline + version
POST /territories → create with auto-fix + version
GET /territories/:id/versions → list boundary versions
POST /territories/:id/restore → restore to a previous version

postgis-helpers.ts:
  validateAndClip(geojson, congregationId) → clipped geometry
  detectOverlaps(territoryId, geojson) → overlap list
  clipToNeighbors(geojson, territoryId) → neighbor-clipped geometry
B

Client-Side Turf.js + Server Validation

Use Turf.js in the browser for real-time clip/overlap preview during editing, then validate server-side on save. Two-pass: client clips locally for instant feedback, server confirms and persists.

Pros

  • Instant client-side preview without API round-trip
  • Turf.js has good TypeScript types
  • Visual feedback while dragging vertices

Cons

  • Turf.js polygon clipping has known edge cases (self-intersections, multi-polygons)
  • Client/server geometry mismatch risk — Turf vs PostGIS may produce different results
  • +120KB bundle size (Turf.js)
  • Must duplicate validation logic in two places
  • Congregation boundary + all neighbor polygons must be loaded client-side
