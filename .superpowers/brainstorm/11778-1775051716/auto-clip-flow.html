Step-by-Step Save Flow

What happens when user saves a polygon (new or edited)
1 User draws/edits polygon → clicks Save
2 Server runs auto-fix pipeline: ST_MakeValid → ST_Difference(water) → ST_Intersection(congregation) → ST_Difference(neighbors)
3 Server returns preview — original vs auto-clipped geometry + list of changes applied
4 Client shows preview overlay on map: original in red dashed, clipped in green solid
5 User sees what changed + can Accept or Cancel
6 On Accept → server saves clipped geometry + creates version snapshot of previous boundary
7 If no changes needed (no violations) → saves directly, no preview dialog

Polygon Versioning

TerritoryBoundaryVersion model
model TerritoryBoundaryVersion {
id String @id @default(uuid())
territoryId String // FK → Territory
version Int // auto-increment per territory
boundaries Json // GeoJSON snapshot
changeType String // "manual_edit" | "auto_clip" | "import" | "creation"
changeSummary String? // "Clipped to congregation boundary, removed overlap with T-302"
createdAt DateTime @default(now())
}

Restore flow: Territory detail page shows version history dropdown. User picks a previous version → preview on map → confirm restore → creates a new version (restore is non-destructive).

Batch Fix for Existing Territories

How should the bulk cleanup work?

A

"Fix All Violations" button on Map View

One-click button in the Kartenansicht toolbar. Runs auto-clip on ALL territories that violate congregation boundary or overlap neighbors. Shows a summary preview: "12 territories will be clipped, 5 overlaps resolved" → Confirm → batch save with versions.

B

Per-territory "Fix" badge on Map View

Each violating territory gets a red warning icon on the map. Click the icon → shows preview of auto-clip for that specific territory → Accept/Cancel. Fix one at a time, but with visual indicators showing which territories need attention.

C

Both — bulk button + per-territory badges

Warning badges on each violating territory (click to fix individually). Plus a "Fix All" button in the toolbar for bulk resolution. Best flexibility but more UI work.
