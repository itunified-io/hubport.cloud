User opens app offline but update is required
📡
Update required — connect to internet
You have 3 unsent changes. Connect to sync and update.
... app remains usable in read-only mode (cached data) ...
New mutations blocked until update completes

Update Sequence (what happens on "Update Now")

Step-by-step:
1. Check navigator.onLine — if offline, show "connect first" and abort
2. Push all pendingChanges to server (POST /sync/push)
3. Resolve any conflicts (conflict UI if needed)
4. If Dexie schema version changed: db.delete() + recreate with new schema
5. Send skipWaiting() to waiting service worker
6. window.location.reload() — new SW takes over
7. App boots with new code → triggers fresh full sync from server

iOS Safari note: skipWaiting() works reliably on iOS. The reload ensures the new SW is controlling the page. No clients.claim() race condition because we reload explicitly.

Vite Config Changes

Current: registerType: "autoUpdate" + skipWaiting: true
→ SW auto-activates without user consent

New: registerType: "prompt" + skipWaiting: false
→ SW stays in "waiting" state until app explicitly calls skipWaiting()
→ useRegisterSW() hook provides needRefresh + updateServiceWorker()

Version injection:
→ Build injects __APP_VERSION__ into bundle (from package.json)
→ Compared against minClientVersion from GET /sync/status
→ Determines optional vs. required update mode
