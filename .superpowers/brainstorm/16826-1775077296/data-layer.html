Encryption Strategy

AES-256-GCM via Web Crypto API
Key Derivation
1. OIDC access_token + deviceId → HMAC-SHA256 → seed
2. seed → PBKDF2 (100K iterations, random salt) → AES-256 key
3. Salt stored in syncMeta (not encrypted)
4. Key held in memory only — never persisted
What Gets Encrypted
● PII fields: names, addresses, notes, phone numbers
● GeoJSON boundaries (location data)
● Pending change payloads
○ IDs, timestamps, status enums — not encrypted (needed for indexing)
iOS Note: Each encrypted field gets a unique IV (12 bytes, crypto.getRandomValues). Dexie middleware intercepts put/get — transparent to app code. Web Crypto SubtleCrypto available on iOS Safari 15+.
