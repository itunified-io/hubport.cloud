Offline Architecture — 3 Approaches

Each approach handles the same requirements differently. Click the one that fits best.

A

Encrypted IndexedDB + Custom Sync Engine

Stack: Raw IndexedDB + Web Crypto API + custom sync logic

Build everything from scratch. IndexedDB stores encrypted blobs per table. Custom diff-based sync compares local timestamps vs server. Conflict detection via version counters.

Pros

  • Zero dependencies — smallest bundle
  • Full control over encryption & sync
  • No library lock-in

Cons

  • Most development effort (weeks)
  • Must handle all IndexedDB edge cases (iOS quirks, quota, transactions)
  • Conflict resolution from scratch
  • Migration/schema changes are manual

⏱ Estimated: 3-4 weeks

B

Dexie.js + Encryption Middleware + Sync Protocol

Stack: Dexie.js (IndexedDB wrapper) + dexie-encrypted + custom sync

Dexie provides a clean Promise-based API over IndexedDB with schema versioning and migrations. Encryption via middleware that transparently encrypts/decrypts on read/write. Sync engine built on top using Dexie's hooks.

Pros

  • Battle-tested IndexedDB wrapper (500K+ weekly npm downloads)
  • Schema versioning & migrations built-in
  • Transparent encryption middleware
  • Rich query API (compound indexes, where clauses)
  • iOS Safari well-tested by community

Cons

  • ~40KB bundle addition (gzipped)
  • Sync engine still custom-built
  • Dexie Cloud exists but is SaaS — we'd use Dexie locally only

⭐ Recommended — best balance of effort vs control

⏱ Estimated: 1.5-2 weeks

C

RxDB + CRDTs

Stack: RxDB (reactive database) + built-in replication + encryption plugin

Full offline-first database with built-in replication protocol, encryption plugin, and reactive queries. RxDB handles conflict resolution via revision trees (CouchDB-style).

Pros

  • Replication protocol built-in (pull/push/conflict)
  • Encryption plugin included
  • Reactive queries (Observable-based)
  • Most feature-complete solution

Cons

  • Heavy — 150KB+ gzipped, RxJS dependency
  • Requires backend replication endpoint (custom protocol)
  • Opinionated — may fight with existing Fastify/Prisma patterns
  • Premium features behind paid license
  • Overkill for our data volume

⏱ Estimated: 2-3 weeks (including backend replication endpoint)
