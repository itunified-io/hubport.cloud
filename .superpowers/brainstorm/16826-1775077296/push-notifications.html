iOS-Specific Requirements

What's needed for Web Push on iOS Safari
✅ Must be installed (Add to Home Screen)
Push only works for installed PWAs on iOS
Browser tab ≠ installed PWA
Show install guide if not installed
✅ iOS 16.4+ required
Detect iOS version via user-agent
If < 16.4: hide notification option
Show "Update iOS for notifications"
⚠️ Permission must be user-triggered
iOS rejects programmatic permission requests
Must be called from user gesture (tap/click)
Our in-app banner approach satisfies this
⚠️ No silent push
Every push must show a visible notification
Can't use push to trigger background sync
"Data-only" push not supported on iOS

Push Subscription — Backend

model PushSubscription {
  id          String   @id @default(uuid())
  tenantId    String
  deviceId    String   @unique  // links to Device
  endpoint    String             // browser push endpoint URL
  p256dh      String             // encryption key (encrypted in DB)
  auth        String             // auth secret (encrypted in DB)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  device      Device   @relation(fields: [deviceId], references: [id], onDelete: Cascade)
  @@index([tenantId])
}
• Cascade delete: revoking a device auto-deletes its push subscription
• p256dh and auth encrypted at rest (ADR-0082 field encryption)
• One subscription per device (1:1 with Device)
