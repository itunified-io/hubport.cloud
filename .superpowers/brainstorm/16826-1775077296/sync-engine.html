Section 2: Sync Engine

Pull/push protocol, conflict resolution, and iOS-safe sync lifecycle

Sync Protocol — Delta-Based
↓ PULL — Server → Client
GET /sync/pull?since=2026-04-01T10:00:00Z

Server queries all syncable tables for updatedAt > since
Returns delta payload grouped by table:
{
  "serverTime": "2026-04-01T12:30:00Z",
  "tables": {
    "territories": { "upserts": [...], "deletes": ["id1"] },
    "addresses":   { "upserts": [...], "deletes": [] },
    "visits":      { "upserts": [...], "deletes": [] },
    "assignments": { "upserts": [...], "deletes": [] },
    "meetings":    { "upserts": [...], "deletes": [] },
    "meetingPoints": { "upserts": [...], "deletes": [] }
  },
  "hasMore": false
}
Pagination: If delta > 500 records, hasMore: true — client calls again with cursor until complete
↑ PUSH — Client → Server
POST /sync/push

Client sends all pending changes from pendingChanges table:
{
  "deviceId": "uuid-...",
  "changes": [
    {
      "table": "addresses",
      "recordId": "addr-123",
      "operation": "update",
      "version": 3,
      "payload": { "status": "not_at_home", "notes": "..." }
    },
    {
      "table": "visits",
      "recordId": "visit-new-1",
      "operation": "create",
      "version": 0,
      "payload": { "addressId": "addr-123", "outcome": "contacted", ... }
    }
  ]
}
⚡ PUSH Response — Per-Change Status
{
  "results": [
    { "recordId": "addr-123", "status": "conflict",
      "serverVersion": 5,
      "serverData": { "status": "do_not_call", "notes": "..." },
      "clientVersion": 3 },
    { "recordId": "visit-new-1", "status": "accepted",
      "serverVersion": 1 }
  ]
}
Statuses: accepted · conflict · rejected (validation error)

Version Tracking — Backend

Prisma schema addition (all syncable models): 
model Address {
  ...existing fields...
  version    Int      @default(0)
  updatedAt  DateTime @updatedAt
  deletedAt  DateTime?  // soft-delete for sync
}
• Every mutation increments version automatically via Prisma middleware
• Soft-delete (deletedAt) instead of hard delete — sync needs to propagate deletes
• Push endpoint: WHERE version = clientVersion — 0 rows updated = conflict
