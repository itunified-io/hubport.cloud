Design Section 3: Snap-to-Road System

Magnet snap while dragging + batch "Snap All" for existing polygons

Snap Architecture

CLIENT SIDE
PolygonEditor enters edit mode
↓ fetches snap context for map bbox
SnapEngine (pure function)
• Builds spatial index from road/building/water geometries
• snapPoint(lngLat, threshold) → nearest point on nearest feature
• snapAll(vertices, threshold) → snapped vertices + report
↓ updates vertex positions in real-time
SERVER SIDE (existing)
GET /territories/snap-context
Already exists! Queries Overpass for:
• Roads (all highway types)
• Buildings (with addresses)
• Water bodies (rivers, lakes)
• Local OSM features from DB
↓ returns GeoJSON FeatureCollection
No new backend work needed for snap.
Just consume existing endpoint on frontend.

Magnet Snap (While Dragging)

Vertex being dragged near a road

Algorithm: On each drag frame, snapPoint() finds the nearest point on any road/building/water LineString within the threshold (30m default). Projects the vertex onto the nearest line segment using perpendicular projection. Shows ghost of original position + blue snap line + distance label.

"Snap All" Batch Operation

Preview: Snap All Vertices to Roads
✓ Accept Snap
✕ Revert

Shows before (orange dashed) vs after (green solid) overlay. Each vertex shows a gray line to its snapped position. User clicks "Accept Snap" to apply or "Revert" to undo. Vertices that couldn't find a nearby feature within threshold stay in place.
