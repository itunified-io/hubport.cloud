Architecture Approach

How to structure the territory editor enhancement — one approach, three layers

Recommended: Unified Inline Editor

Enhance the existing TerritoryDetail map with a proper inline polygon editor. One cohesive system that handles creation, editing, and snapping — no separate pages or components.

1

Smart Creation Flow

New territory from map: User draws rough circle/polygon on the TerritoryMap page (with neighbors + congregation boundary visible). On close:

  • Reverse geocode centroid via Overpass → get city/village name
  • Match city to existing group prefix (e.g., other "Antdorf" territories are 5xx)
  • If new city, suggest next available prefix
  • Show modal: pre-filled city name + suggested prefix, user picks exact number
  • Save → navigate to detail page with polygon already set
2

Inline Polygon Editor

Edit mode on detail page: Click "Edit" → polygon vertices become draggable circles on the map.

  • Drag vertex — move to new position (magnet-snap to nearby roads if within 30m)
  • Click midpoint — insert new vertex between two existing ones
  • Right-click vertex — delete it (min 3 vertices)
  • Snap toggle — enable/disable magnet snap while dragging
  • Save/Cancel toolbar — appears at top, runs auto-fix pipeline on save
  • Neighboring territories shown as translucent overlays during edit
3

Snap-to-Road System

Two snap modes using the existing /territories/snap-context endpoint:

  • Magnet snap (while dragging) — as user drags a vertex within 30m of a road, vertex "magnetizes" to nearest point on road geometry. Visual indicator (blue highlight on road segment).
  • "Snap All" button — batch-snap all vertices to nearest roads. Shows before/after preview overlay (orange = original, green = snapped). Accept or cancel.
  • Snap targets: roads, building outlines, water boundaries (from existing Overpass queries)
  • Configurable threshold (30m default, adjustable)
