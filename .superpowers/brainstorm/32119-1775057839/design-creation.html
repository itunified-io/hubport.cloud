Design Section 1: Smart Creation Flow

Draw on map → auto-detect city → pick number → save

User Flow

Step-by-Step Flow
STEP 1
Click "+ New Territory" on map page
Map shows all existing territories + congregation boundary (red dashed). No modal — goes straight to drawing mode.
STEP 2
Draw rough polygon
Click vertices or freehand lasso. Neighboring territories and congregation boundary remain visible underneath. Double-click or Enter to close.
STEP 3 (auto)
System auto-detects
1. Reverse geocode centroid → city name (e.g., "Antdorf")
2. Find existing territories with same city → determine group prefix (e.g., 5xx)
3. Find next available number in that group (e.g., 507)
STEP 4
Confirm in modal
Modal shows:
• Pre-filled name: "Antdorf" (editable)
• Suggested number: "507" (editable)
• Group hint: "5xx — Antdorf (501, 503, 505 exist)"
• Create button → saves with drawn polygon

Backend: Reverse Geocode Endpoint

POST /territories/suggest

// Request
{ boundaries: GeoJSON Polygon }

// Response
{
  city: "Antdorf",
  suggestedPrefix: "5",
  suggestedNumber: "507",
  existingInGroup: ["501", "503", "505"],
  autoFix: { ...AutoFixResult }
}

Uses Overpass is_in query on centroid to get admin boundary name. Matches against existing territory names to find group prefix. Runs auto-fix pipeline on the polygon. All in one round-trip.

Key Change: Drawing Happens on TerritoryMap (not TerritoryDetail)

Currently: "+ New Territory" opens a modal asking for number + name, then navigates to detail page to draw.
New: "+ New Territory" enters drawing mode directly on the map view — user sees all neighbors and the congregation boundary while drawing. After drawing, modal asks for number + name (pre-filled). Territory is created with the polygon in one step.
